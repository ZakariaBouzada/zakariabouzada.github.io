Project Overview

This project demonstrates an AI-powered assistant integrated into a personal portfolio website. Visitors and recruiters can ask questions about Zakaria's skills, projects, and background, and receive accurate, concise responses in real time.

Objective

To showcase capabilities in backend API development, cloud deployment, prompt engineering, and AI integration — providing an interactive demonstration of full-stack AI functionality for visitors and recruiters.

Model and API

  • Model: Qwen2.5-7B-Instruct — a 7 billion parameter instruction-tuned language model by Alibaba Cloud, optimized for chat and instruction following.
  • Provider: Served via the Hugging Face Router API at https://router.huggingface.co/v1/chat/completions, automatically routing to the fastest available inference provider.
  • Key Features: Conversation history (last 5 exchanges), system prompt with strict facts, few-shot examples for consistent response style, max token limit set to 150.
  • Security: API key stored server-side in a .env file on Render — never exposed to the frontend. User data is not persisted.

Technical Architecture

Built with Python Flask, the backend handles incoming requests from the frontend, maintains conversation history, and forwards structured message payloads to the Hugging Face Router API. The backend is deployed on Render for continuous availability with a server wake-up ping on page load to minimize cold start delays.

Technical Design

Interactive chat implemented with vanilla HTML, CSS, and JavaScript — no frameworks. The frontend sends asynchronous fetch requests to the Flask backend, passing the user question and full conversation history with each request. Responses stream back into a dynamic chat window. A mini version of the chat is embedded directly on the homepage header for immediate engagement.

Prompt Engineering

The assistant uses a structured system prompt combining strict behavioral rules, a curated FACTS section derived from Zakaria's CV, and few-shot examples to enforce concise and accurate responses. This approach ensures the model stays on-topic and avoids hallucination without requiring fine-tuning.

  • Model: Qwen/Qwen2.5-7B-Instruct on Hugging Face

Challenges and Solutions

  • Used the model Llama 3.2-1B-Instruct, but the model was too small and didn't follow instructions, so had to find a free, fast and reliable model to replace it
  • Hugging Face deprecated api-inference.huggingface.co — solution: migrated to router.huggingface.co with provider routing.
  • Handling CORS for GitHub Pages — solution: enabled Flask-CORS in the backend.
  • Model hallucination and rambling — solution: strict system prompt rules, reduced max tokens, lowered temperature to 0.3, and few-shot prompting.
  • Cold start latency on Render free tier — solution: server wake-up ping sent automatically on page load.
  • Mini chat integration on homepage — solution: compact embedded chat with suggested question chips for immediate engagement.

Learning Outcomes

  • Integrating third-party LLM APIs and managing API keys securely in production.
  • Prompt engineering techniques — system prompts, few-shot examples, temperature tuning.
  • Deploying and maintaining a Python Flask backend on a cloud platform.
  • Building conversational UI with stateful history in vanilla JavaScript.
  • Navigating real-world API deprecations and provider migration in a live deployment.
