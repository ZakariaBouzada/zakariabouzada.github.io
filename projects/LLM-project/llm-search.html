Project Overview

This project demonstrates a mini AI-powered search engine integrated into a personal portfolio website. Users can ask questions about my skills, projects, or experiences, and the AI provides responses in real time.

Objective

To showcase capabilities in backend API development, cloud deployment, and AI integration, providing an interactive and visually appealing demonstration of AI functionality for visitors and recruiters.

Model and API

  • Model: Meta LLaMA 3.2 (1B Instruct) — a state-of-the-art large language model.
  • API: Hugging Face Inference API via https://router.huggingface.co/v1/chat/completions.
  • Key Features: Chat-style responses, system instructions, max token limit set to 500.
  • Security: Requests sent via API key stored in a .env file; user data is not persisted externally.

Technical Architecture

Built with Python Flask, the backend handles incoming requests from the frontend, forwards them to the Hugging Face API, and returns AI-generated answers. The backend is deployed on Render for continuous availability.

Technical Design

Interactive chat implemented with HTML, CSS, and JavaScript. Users can type questions into a text box, and answers appear in a dynamic chat window above the input. The frontend communicates with the backend using asynchronous fetch requests.

Deployment & Links

  • Backend URL: https://llm-search-zakariabouzada-github-io.onrender.com
  • Frontend: Hosted via GitHub Pages.

Challenges and Solutions

  • Hugging Face deprecated api-inference.huggingface.co — solution: migrated to router.huggingface.co.
  • Handling CORS for GitHub Pages — solution: enabled Flask CORS in the backend.
  • Mini chat integration on homepage — solution: added a compact search bar with pop-up chat responses.

Learning Outcomes

  • API integration and managing API keys securely.
  • Deploying a backend service for public access.
  • Building an interactive frontend chat interface.
  • Documenting technical projects in a clear, recruiter-friendly manner.
